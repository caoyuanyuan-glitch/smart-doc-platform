# 说明书审核能力补强方案

## 背景

针对英文说明书人工审核意见与平台审核结果的对比，当前审核模块已能较好发现单位、标点、拼写和部分术语问题，但对跨章节一致性、说明书结构、型号覆盖、表图编号、货号一致性、操作步骤语气等工程化审核项覆盖不足。

后续审核能力建设应将人工审核经验沉淀为可执行规则、结构化抽取和专项 AI 复核流程。

## 人工审核意见类型归类

### 版本记录

- Revision History 中最新版本记录应置于最上方。
- 版本号、日期和修订说明格式应统一。

### 适用范围一致性

- 封面、适用软件版本、硬件配置、流程说明中涉及的设备范围应一致。
- 若文档声明适用于 MGISP-960 和 MGISP-NE384，硬件配置、操作章节和流程说明中应同时覆盖对应设备。

### 型号与产品名一致性

- 型号应按白名单检查，例如 MGISP-960、MGISP-NE384、MGISP-NE384RS、MGISTP-7000。
- 对相似拼写和漏字错字进行检查，例如 MGI-SP960、MGIISP；MGISTP-7000 属于白名单型号，不应改为 MGISP-7000。
- 同一设备名称在全文中应保持一致。

### 表图编号连续性

- Table 和 Figure 编号应按章节连续。
- 应检查缺号、跳号、重复编号和标题编号不一致。
- 表图引用编号应能对应到实际表图。

### 货号一致性

- 同一试剂、耗材或组件在不同表格中的 Cat. No. 应一致。
- 表格内 Cat. No. 写法应统一为 `Cat. No.`。
- 跨表出现同名物料时，应比对规格、数量和货号。

### 章节结构

- 内容过少的孤立小节应提示合并或补充内容。
- 章节标题层级应合理，避免小节只包含一句话或无法独立成节的内容。

### 操作步骤语气

- 操作步骤应优先使用祈使句。
- 操作步骤应使用一般现在时。
- 应避免过去时、被动语态和叙述式表达。

### 英文文档中文混入

- 英文说明书中不应残留中文字符、中文标点、中文单位或中文说明。
- 中文标点应替换为英文半角标点。

### 文案规范

- 检查大小写统一、逗号缺失、分隔符缺失、多余字符、冗余词和不清晰表达。
- 对 `the`、`any`、`foil` 等人工标注词，需要结合上下文判断具体修改意图。

## 能力建设方案

### 结构化规则优先实现

以下规则应优先用代码规则实现，以保证稳定性和可回归测试：

- Revision History 最新版本置顶检查。
- 英文文档中文字符和中文标点混入检查。
- 型号白名单与近似拼写检查。
- Table/Figure 编号连续性检查。
- `Cat. No.` 写法规范检查。
- 单位、标点、空格、大小写基础规范检查。
- 操作步骤祈使句和时态初筛。
- 超短小节识别。
- 货号跨表一致性检查。

### 跨章节一致性抽取后比对

跨章节问题需要先抽取结构化信息，再进行一致性校验：

- 适用设备：MGISP-960、MGISP-NE384、NE384 等。
- 软件版本：例如 1.6.1.301。
- 硬件配置表：硬件名、数量、位置、适用设备。
- 试剂耗材表：名称、规格、Cat. No.。
- 操作台面位置：Pos1 到 PosN。

一致性校验示例：

- 适用范围提到 NE384 时，硬件配置和操作流程应覆盖 NE384。
- 同一耗材名称在不同表格中的 Cat. No. 应一致。
- 操作章节中的台面位置编号应能对应硬件配置章节。
- 封面、正文、表格中的设备型号应保持一致。

### LLM 专项复核

LLM 应作为专项复核工具使用，每次只处理一个明确主题和相关上下文：

- `structure_review`：章节结构、短小小节、版本记录顺序。
- `model_consistency_review`：型号、平台、适用范围一致性。
- `procedure_language_review`：操作步骤祈使句、一般现在时、被动语态。
- `table_consistency_review`：表格货号、位置编号、硬件配置一致性。
- `human_style_review`：表达不清、需要确认、上下文依赖较强的问题。

LLM 输出必须使用固定 JSON 结构，包含问题类型、位置、原文、建议、判断依据和置信度。

## 回归测试要求

应将带人工 PDF 注释的说明书作为回归测试集：

- 提取 PDF 注释作为人工审核金标准。
- 每次修改审核逻辑后，自动运行同一 PDF。
- 输出人工问题覆盖率、漏检项、多报项和误报项。
- 初期目标为人工问题覆盖率达到 70% 以上，再逐步降低误报。

## 优先级建议

1. 补充英文文档中文混入、表图编号、Revision History、型号白名单、Cat. No.、操作步骤语气规则。
2. 实现硬件配置、耗材货号、台面位置的结构化抽取和跨表一致性校验。
3. 将 LLM 深度审核拆分为专项复核流程。
4. 建立人工注释 PDF 回归评测集，持续跟踪覆盖率和误报率。
