**版本**：V1.0  
**更新日期**：2026-06-15  
**作者**：技术文档智能润色助手  
**适用范围**：自动化设备说明书、IVD 产品说明书、系统操作指南等技术文档

---

## 1. 功能概述

本 Skill 用于对 Word 技术文档（`.docx`）进行标准化润色，以句式表达参考手册和术语库为基准，通过整段替换方式修正病句、统一术语、规范标点与格式，并生成带修订标记的润色版本及润色报告。

---

## 2. 启动条件

用户发送以下任一指令时触发：
- "开始润色"
- "润色文档"
- "帮我润色"
- "请润色"
- 类似语义

---

## 3. 必需输入信息（5 项）

回复以下模板，收齐后方可开工：

```
请提供以下信息：
- 句式表达标准文件路径：
- 术语库路径：
- 待润色文件路径：
- 输出文件路径：
- 润色要求：
```

### 输入说明

| 字段 | 说明 | 格式 |
|---|---|---|
| 句式表达标准文件路径 | 句式参考手册 docx 的飞书文件链接或本地路径 | 飞书 URL / 本地绝对路径 |
| 术语库路径 | 术语对照表 docx 或飞书链接 | 飞书 URL / 本地绝对路径 |
| 待润色文件路径 | 需要润色的源 docx 文件 | 飞书 URL / 本地绝对路径 |
| 输出文件路径 | 润色结果存放的飞书文件夹 | 飞书 folder URL |
| 润色要求 | 特殊要求或重点关注项 | 自由文本（可选） |

---

## 4. 输出标准

润色任务完成后，必须交付 **2 份文件**，均上传到用户指定的输出文件夹。

### 4.1 【修订标记版】xxx.docx

- **格式**：保留原 docx 格式（标题层级、表格、图片、字体、对齐、段落样式、缩进、注意框）
- **修订标记**：使用 Word 标准 OOXML `<w:ins>` / `<w:del>` 标记所有改动
- **修订作者**：`技术文档智能润色助手`
- **技术实现**：`python-docx + lxml`，整段替换采用 `wrap_run_as_del` + 追加 `ins`
- **查看方式**：Microsoft Word / WPS 桌面端【审阅】→【修订】；飞书在线预览**不显示**修订标记
- **获取干净稿**：Word 中【审阅】→【接受】→【接受所有修订】

### 4.2 【润色报告】xxx.docx

报告内容必须包含以下章节：

| 章节 | 内容 |
|---|---|
| 1. 文档基本信息 | 原文件名、段落数、表格数、文件大小、日期 |
| 2. 润色基准 | 句式标准 / 术语库引用文件名 + 链接 |
| 3. 改动统计 | 规则总数、实际改动数、段落命中数、修订标记类型 |
| 4. 主要润色方向分类表 | 问题类型 + 修复举例（病句修复、标点符号、中英文混排、术语统一等） |
| 5. 完整改动清单 | 序号 / 段落定位 / 修改前 / 修改后 / 改动类型 |
| 6. 查看修订的方法 | Word/WPS 操作步骤、飞书在线预览不支持提示 |

## 5. 润色规则体系

### 5.1 句式规范化

以句式表达参考手册为标准，对以下类型进行规范化：

| 类型 | 标准句式示例 |
|---|---|
| 安全警告 | 禁止…。/ 请勿…，以防… / 必须…，否则… |
| 操作步骤 | 点击【XX】。/ 选择【XX】下拉列表，选择… / 将…放置到… |
| 条件分支 | 如…，请… / 若…，则… / 当…时，… |
| 建议推荐 | 建议… / 推荐… / 强烈建议… |
| 数值规格 | 约… / 至少… / 在…范围内 / 温度为… |
| 注意事项 | 注意：… / 警告：… / 提示：… |
| 设备状态 | 本仪器… / 当前状态为… / 指示灯…表示… |
| 维护保养 | 清洁… / 擦拭… / 每次… / 每日… / 定期… |
| 故障处理 | 异常…，请… / 如仍…，请联系… / 报错代码：… |
| 责任声明 | 仅限… / 仅供… / 未经授权… / 对…不做任何保证 |
| 储存运输 | 储存条件：… / 运输温度：… / 直立放置。 |
| 文档引用 | 见图… / 详见… / 参见… |

### 5.2 术语统一

以术语库为标准，对全文进行术语替换：

| 不规范用词 | 规范用词 |
|---|---|
| 建库仪 | 本仪器 |
| 产品组成 | 本产品由…组成 |
| 该软件 | 本控制软件 |
| 电源电缆 | 电源线 |
| 枪盒 | 吸头盒 |
| 枪头 | 吸头 |
| 八连管 | 八联管 |
| 试验台 | 实验台 |
| PCR模块 | PCR 模块 |
| 200μL | 200 μL |
| 4℃ | 4 ℃ |

### 5.3 格式规范

1. **中英文/数字间空格**：数字与单位、英文缩写与中文间加空格
2. **段尾标点**：段尾统一用句号，不用分号或省略
3. **UI 控件格式**：界面按钮/菜单用【】包裹，如【确定】、【菜单】
4. **占位符**：原文中的 tab/图标用文字占位符替代，如 `[图标]`、`[取消]`
5. **章节引用**："本文05章" → "本文第 5 章"

## 6. 工作流程

```
用户发送"开始润色"
    ↓
回复模板，收集 5 项信息
    ↓
下载源文件（句式标准 + 术语库 + 待润色文件）
    ↓
分析文档结构（段落/表格/字数统计）
    ↓
逐段匹配规则（old → new 整段替换）
    ↓
生成【修订标记版】docx（OOXML ins/del）
    ↓
生成【润色报告】docx（统计 + 清单 + 方法）
    ↓
上传 2 份文件到飞书输出文件夹
    ↓
回复用户完成摘要 + 下载链接
```

## 9. 限制与说明

1. **输入格式**：优先 `.docx`；`.pdf` 需先提取文本，无法保留原排版做修订标记
2. **旧版 `.doc`**：需用户先转换为 `.docx`（环境无 LibreOffice）
3. **飞书在线预览**：不显示 OOXML 修订标记，必须下载到 Word/WPS 桌面端查看
4. **图片中文字**：需先 OCR，不在本 Skill 支持范围内
5. **修订标记数量**：超过 500 处时 Word 可能卡顿，建议分批处理
