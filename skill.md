# 英文文档综合检查小程序 - 技术&使用文档

## 1. 基础信息
- 项目名称：英文文档拼写、格式、语法离线检查工具
- 开发语言：Python
- GUI框架：Tkinter（Python内置）
- 运行特性：**纯离线运行，无需联网**，可跨电脑分发使用
- 支持文档格式：`.txt`、`.docx`(Word)、`.xlsx`(Excel)、`.pptx`(PPT)、`.dita`、内含dita文件的`.zip`压缩包
- 核心作用：面向英文技术文档，自动检测拼写、格式、主谓语法错误，支持错误快速遍历与定位

---

## 2. 运行环境 & 依赖组件

### 2.1 基础环境
- 操作系统：Windows / macOS / Linux
- Python 版本：Python 3.8 及以上（推荐 3.9 ~ 3.11）

### 2.2 第三方依赖库
打开命令提示符(CMD)/终端，执行以下命令批量安装依赖：
```bash
pip install pyenchant python-docx openpyxl python-pptx lxml
```
依赖库作用说明：
1. `pyenchant`：英文拼写检查核心库，依赖系统Enchant词典引擎
2. `python-docx`：解析读取 Word (.docx) 文档
3. `openpyxl`：解析读取 Excel (.xlsx) 文档
4. `python-pptx`：解析读取 PPT (.pptx) 文档
5. `lxml`：XML解析库，用于读取 DITA 格式文件

### 2.3 系统额外依赖（Windows重点）
若拼写功能异常，需单独安装 **Enchant 拼写引擎**，否则单词校验失效。

## 3. 整体工作流程
1. **启动程序**：运行 Python 脚本，打开可视化GUI窗口，初始化规则、界面、错误列表。
2. **加载文档**：点击「打开文件/ZIP」选择本地文件，程序自动解析提取文本并展示；
   内置规则：**连续3行及以上空行自动忽略，不加载、不参与检查**。
3. **一键全量检查**：触发全维度错误检测，收集所有错误并按文档字符位置排序。
4. **可视化标记**：不同错误用背景色区分，底部状态栏统计错误数量。
5. **错误遍历定位**：使用「上一处错误/下一处错误」按文档顺序逐个跳转定位。
6. **查看详情**：鼠标点击标记区域，弹窗展示具体错误类型。
7. **词典管理（可选）**：添加专业词汇、导入/导出词典，规避专业名词误判。
8. **循环使用**：可重复加载新文档、重新检测。

## 4. 全部功能明细

### 4.1 文件解析功能
1. 多格式兼容：支持 TXT、Word、Excel、PPT、DITA、ZIP(仅含DITA)。
2. 异常容错：读取失败、编码异常自动捕获，弹窗提示，程序不崩溃。
3. 空行过滤：连续≥3行空行直接跳过，不做检查。
4. 文本预览：解析后的完整文本展示在编辑区域。

### 4.2 错误检测规则（核心校验）

#### 4.2.1 拼写错误（黄色背景标记）
- 基于英文词典校验单词拼写，识别错拼词汇。
- 支持自定义词典，专业术语添加后不再误报。

#### 4.2.2 格式类错误（浅蓝色背景标记）
- 多余空格：连续2个及以上空格判定为错误。
- 标点前多余空格：标点符号前方存在无效空格。
- 标点后缺失空格：单词与标点之间未正常分隔。
- 句首大小写错误：句子开头字母小写判定为错误。

#### 4.2.3 主谓不一致语法错误（浅蓝色背景标记）
1. There be 句型：严格遵循**就近原则**，根据be动词后第一个有效名词单复数校验。
2. 常规主谓校验：仅校验合法主语 + be动词/助动词；仅校验 he/she + 实义动词三单形式。
3. 全局防误判黑名单（自动豁免，不判定主谓错误）
   - 技术名词：guide、system、interface、chapter、section、page、tab、range、device、moment、step、sample、problem、air、equipment、performance、case、U.S.
   - 介词：in、to、on、up、into、with、at、for、by、from
   - 副词：still、briefly、downward、no、now、here、there、also
   - 代词/特殊词：you、each、it、what、do
4. 统一豁免规则
   - 所有冠词（the/a/an）不参与主语校验。
   - 情态动词（can/could/may/might/will/would/shall/should/must）跳过语法校验。

### 4.3 错误遍历与定位
1. 排序规则：**所有错误（拼写+格式+语法）按文档从上到下、从左到右的字符位置统一排序**。
2. 全类型遍历：「上一处/下一处错误」遍历全部黄标、蓝标错误，无遗漏、不分类遍历。
3. 循环跳转：遍历至末尾自动回到第一条，开头自动跳转至最后一条。
4. 自动滚动：定位错误时，文本区自动滚动到可视位置。
5. 详情查看：点击标记区域，弹窗展示错误具体描述。

### 4.4 离线词典管理
1. 添加单词：手动录入专业词汇，加入本地词典，屏蔽拼写误报。
2. 词典模板：生成标准txt模板，用于批量整理词汇。
3. 导入词典：读取外部txt文件，批量加载自定义词汇。
4. 导出词典：将本地自定义词汇导出为txt，用于备份、分发。

### 4.5 界面辅助功能
1. 双色标记：黄色=拼写错误；浅蓝色=格式/语法错误。
2. 数据统计：底部状态栏实时显示「拼写错误数」「语法/格式错误数」。
3. 离线闭环：所有功能、词典、规则均本地生效，全程无网络请求。

## 6. 核心约束规则汇总
1. 空行规则：连续3行及以上空行 → 直接忽略，不检查。
2. 标记规则：黄色背景=拼写错误；浅蓝色背景=格式/语法错误。
3. 语法豁免：黑名单词汇、冠词、情态动词，均不判定为主谓不一致错误。
4. 遍历规则：所有错误按文档物理位置依次遍历，不区分错误类型。
5. 运行规则：纯离线运行，支持多电脑拷贝使用。

## 7. 简易使用教程
1. 前置准备：安装 Python + 全部依赖库。
2. 启动程序：双击运行 `.py` 脚本文件，打开工具界面。
3. 加载文件：点击【打开文件/ZIP】，选择需要检查的文档。
4. 执行检查：点击【一键检查】，自动标记所有错误并统计数量。
5. 定位错误：使用【上一处错误】【下一处错误】逐个查看问题位置。
6. 处理专业词汇：生僻术语可通过「添加单词/导入词典」加入自定义词典。
7. 备份词典：更换设备前，可使用「导出词典」保存自定义词汇。
8. 关闭程序：直接关闭窗口即可。
