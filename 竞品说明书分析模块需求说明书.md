# 竞品说明书分析模块 需求说明书

## 一、模块概述

### 1.1 模块名称
竞品说明书分析

### 1.2 模块定位
AI驱动的竞品文档与内容情报分析系统，实现竞品说明书（网页版 + 文档版 + 公众号文章）的自动监控、多维度对比分析与智能改进建议输出，帮助产品团队和文档团队及时掌握竞品动态，优化自身文档质量与内容策略。

### 1.3 核心价值
- **感知层**：实时监控竞品多渠道内容更新（官网说明书 + PDF文档 + 公众号文章）
- **分析层**：多维度对比竞品与我方的内容差异、质量差异、策略差异
- **决策层**：AI智能分析并输出可执行的改进建议与行动指南

### 1.4 目标用户
| 用户角色 | 核心需求 |
|---------|---------|
| 产品经理 | 跟踪竞品功能迭代、产品卖点变化、市场动态 |
| 技术文档工程师 | 对标竞品文档结构、内容完整性、写作质量 |
| 市场/运营人员 | 了解竞品营销话术、内容策略、活动动态 |
| 研发工程师 | 关注竞品技术参数、功能规格更新 |
| 内容运营 | 分析竞品公众号内容策略，优化我方内容规划 |

### 1.5 监控渠道矩阵
| 渠道类型 | 内容形式 | 更新频率 | 情报价值 |
|---------|---------|---------|---------|
| 官网说明书页面 | HTML网页 | 低（季度/月度） | ⭐⭐⭐⭐⭐ 最权威 |
| PDF/Word说明书 | 文档下载 | 低（季度/月度） | ⭐⭐⭐⭐⭐ 最完整 |
| 公众号文章 | 图文内容 | 高（周/天） | ⭐⭐⭐⭐ 最及时 |

## 三、详细功能需求

### 模块一：竞品管理

#### 1.1 竞品信息维护
**功能描述**：维护竞品基础信息

| 功能点 | 详细说明 |
|-------|---------|
| 竞品新增 | 支持添加竞品，包含：竞品名称、品牌、官网地址、竞品简介 |
| 竞品编辑 | 修改竞品基本信息 |
| 竞品删除 | 删除竞品及其相关数据（二次确认） |
| 竞品列表 | 分页展示竞品列表，支持按名称搜索、按标签筛选 |
| 竞品详情 | 查看竞品详细信息及关联的所有监控源 |

#### 1.2 监控源配置
**功能描述**：为每个竞品配置需要监控的内容来源（三大渠道）

| 功能点 | 详细说明 |
|-------|---------|
| 网页版源 | 添加网页版说明书URL，支持多个URL |
| 文档版源 | 添加PDF/Word文档下载链接或上传初始版本 |
| 公众号源 | 添加竞品公众号名称/微信号，支持多个公众号 |
| 源类型标记 | 标记每个源的类型：产品介绍、技术参数、使用手册、API文档、品牌公众号、产品公众号等 |
| 监控频率设置 | 为每个源设置监控频率：每小时/每天/每周/每月/手动 |
| 关注区域配置 | 网页版支持配置CSS选择器，只监控页面特定区域 |
| 关键词配置 | 公众号支持配置关注关键词，筛选相关文章 |

#### 1.3 标签与分组管理
**功能描述**：对竞品进行分类管理

| 功能点 | 详细说明 |
|-------|---------|
| 标签管理 | 新增/编辑/删除标签 |
| 打标签 | 为竞品打上多个标签 |
| 分组管理 | 按产品线/竞品类型分组 |
| 筛选查询 | 按标签/分组筛选竞品 |

### 模块二：版本监控

#### 2.1 网页版监控
**功能描述**：自动监控竞品官网说明书页面的变化

| 功能点 | 详细说明 |
|-------|---------|
| 页面抓取 | 使用Playwright模拟真实浏览器，抓取完整渲染后的页面内容 |
| 内容提取 | 自动提取页面正文文本、标题结构、图片、链接 |
| 截图保存 | 保存页面完整长截图，用于视觉对比 |
| DOM结构保存 | 保存完整HTML结构，用于结构变化分析 |
| 定时抓取 | 按配置的频率自动执行抓取任务 |
| 手动触发 | 支持手动触发立即抓取 |
| 动态内容处理 | 处理需要滚动加载、点击展开的内容 |

#### 2.2 文档版监控
**功能描述**：监控竞品PDF/Word等格式说明书的更新

| 功能点 | 详细说明 |
|-------|---------|
| 下载链接监控 | 监控指定URL的文档是否有新版本 |
| 自动下载 | 检测到更新后自动下载新文档 |
| 文档解析 | 解析PDF/Word/DITA等格式文档内容 |
| 版本识别 | 自动识别文档版本号/发布日期 |
| 手动上传 | 支持手动上传竞品文档版本 |

#### 2.3 公众号监控
**功能描述**：监控竞品公众号的文章更新

| 功能点 | 详细说明 |
|-------|---------|
| 公众号添加 | 输入公众号名称/微信号，添加监控 |
| 文章自动采集 | 定时抓取公众号最新文章列表与内容 |
| 文章内容提取 | 提取文章标题、正文、封面图、发布时间、作者 |
| 互动数据采集 | 采集阅读量、在看数、点赞数、评论数（如可获取） |
| 关键词筛选 | 按配置的关键词过滤，只保留相关文章 |
| 文章分类 | 自动识别文章类型：产品发布、技术解读、客户案例、行业观点、活动推广等 |
| 手动添加 | 支持手动输入文章URL添加单篇文章 |
| 批量导入 | 支持批量导入文章链接 |

#### 2.4 历史版本管理
**功能描述**：管理和浏览竞品内容的历史版本

| 功能点 | 详细说明 |
|-------|---------|
| 版本列表 | 按时间倒序展示所有历史版本（含网页、文档、文章） |
| 版本详情 | 查看某个版本的完整内容 |
| 版本标签 | 为重要版本打标签（如：v1.0、重大更新、新品发布） |
| 版本备注 | 为版本添加备注说明 |
| 版本对比 | 选择任意两个版本进行对比 |
| 版本时间线 | 可视化展示版本更新时间线，按渠道分色展示 |

#### 2.5 变化通知告警
**功能描述**：当检测到竞品有更新时，及时通知相关人员

| 功能点 | 详细说明 |
|-------|---------|
| 变化检测 | 自动检测新旧版本差异，计算变化量 |
| 阈值配置 | 配置变化量阈值，超过阈值才触发通知 |
| 通知渠道 | 支持邮件、飞书、站内消息等通知方式 |
| 通知内容 | 通知包含：竞品名称、更新时间、变化摘要、查看链接 |
| 通知记录 | 查看历史通知记录 |
| 订阅管理 | 用户可订阅/取消订阅特定竞品或特定渠道的通知 |

### 模块三：对比分析

#### 3.1 竞品版本对比（纵向对比）
**功能描述**：对比同一竞品不同版本的内容，了解变化情况

| 功能点 | 详细说明 |
|-------|---------|
| 版本选择 | 选择两个版本进行对比 |
| 内容变化对比 | 文本内容差异高亮展示（新增/删除/修改） |
| 结构变化对比 | 章节结构变化（新增章节/删除章节/顺序调整） |
| 视觉变化对比 | 页面截图对比，高亮视觉差异区域 |
| 变化统计 | 统计变化量：新增字数、删除字数、修改段落数 |
| 双栏视图 | 左右双栏同步滚动对比 |

#### 3.2 竞品 vs 我方对比（横向对比）
**功能描述**：对比竞品说明书与我方说明书，找出差距

| 功能点 | 详细说明 |
|-------|---------|
| 文档选择 | 选择竞品文档 + 我方文档进行对比 |
| 章节结构对比 | 目录结构对比，标注双方独有章节 |
| 内容覆盖度分析 | 分析双方内容覆盖情况：<br>- 我方独有内容<br>- 竞品独有内容<br>- 双方共有内容 |
| 功能特性对比 | 提取功能描述，对比功能点差异 |
| 技术参数对比 | 自动提取参数表格，逐项对比参数值 |
| 双栏对比视图 | 左竞品/右我方，同步滚动，差异高亮 |
| 差异点列表 | 逐条列出所有差异点，支持筛选和跳转 |

#### 3.3 内容覆盖度分析
**功能描述**：深度分析内容覆盖的完整度

| 功能点 | 详细说明 |
|-------|---------|
| 主题覆盖分析 | 按主题/功能点分析双方覆盖情况 |
| 独有内容识别 | 识别竞品有但我方没有的内容 |
| 优势内容识别 | 识别我方有但竞品没有的内容 |
| 覆盖度评分 | 量化评分双方内容覆盖度 |
| 覆盖度雷达图 | 多维度可视化展示覆盖度对比 |

#### 3.4 技术参数对比
**功能描述**：自动提取并对比技术参数表格

| 功能点 | 详细说明 |
|-------|---------|
| 表格自动提取 | 从PDF/网页中自动识别并提取参数表格 |
| 参数对齐 | 智能匹配相同参数项（处理命名差异） |
| 逐项对比 | 对比每个参数的数值差异 |
| 差异高亮 | 参数差异项用颜色高亮标注 |
| 参数对比表 | 生成统一格式的参数对比表 |
| 导出功能 | 支持导出参数对比表为Excel |

#### 3.5 文档质量对比
**功能描述**：从多个维度对比双方文档质量

| 功能点 | 详细说明 |
|-------|---------|
| 完整性对比 | 信息完整性评分对比 |
| 结构清晰度对比 | 文档结构合理性评分 |
| 可读性对比 | 句式复杂度、易读性评分 |
| 术语一致性对比 | 术语使用一致性分析 |
| 质量雷达图 | 多维度质量对比可视化 |

## 四、用户操作流程

### 流程一：添加竞品并配置监控
```
1. 进入竞品管理 → 新增竞品
2. 填写竞品基本信息（名称、官网、简介）
3. 添加监控源：
   - 网页版：说明书页面URL
   - 文档版：PDF下载链接或上传文档
   - 公众号：公众号名称/微信号
4. 设置各渠道监控频率和关注规则
5. 保存 → 系统开始自动监控
```

### 流程二：查看竞品更新
```
1. 收到更新通知（站内/邮件/飞书）
2. 进入竞品详情 → 动态时间线
3. 点击最新更新 → 查看变化摘要
4. 选择新旧版本对比 → 查看详细差异
5. 查看AI分析的变化要点
```

### 流程三：竞品 vs 我方对比
```
1. 进入对比分析 → 横向对比
2. 选择竞品 + 我方文档
3. 点击开始对比
4. 查看对比报告：
   - 章节结构对比
   - 内容覆盖度分析
   - 技术参数对比
   - 文档质量对比
5. 查看AI改进建议
6. 导出报告
```

### 流程四：公众号文章分析
```
1. 进入公众号分析 → 文章列表
2. 筛选竞品/时间/关键词，找到目标文章
3. 点击文章 → 查看详情与AI分析摘要
4. 进入策略分析 → 查看内容策略洞察
5. 选择多公众号对比 → 生成对比报告
6. 查看我方内容策略建议
```

## 六、非功能需求

### 6.1 性能要求
- 页面抓取响应时间：≤ 30秒（复杂页面≤ 60秒）
- 文档对比耗时：≤ 2分钟（100页以内文档）
- 公众号文章采集：单篇 ≤ 10秒
- 报告生成耗时：≤ 30秒
- 文章AI分析：单篇 ≤ 15秒

### 6.2 数据存储
- 版本历史保留：至少保留最近50个版本
- 公众号文章保留：永久保留（可配置清理策略）
- 截图存储：按版本保存，支持清理旧版本
- 数据备份：每日自动备份

### 6.3 安全要求
- 竞品数据按用户隔离
- 爬虫行为合规，遵守网站robots.txt
- 设置合理的抓取频率，避免对目标网站造成压力
- 公众号文章采集遵守相关平台规定

### 6.4 兼容性
- 支持主流浏览器：Chrome、Firefox、Edge
- 响应式设计，支持不同屏幕尺寸

## 八、技术架构概览

### 8.1 后端技术栈
- **Web框架**：FastAPI（与现有平台一致）
- **页面抓取**：Playwright
- **公众号采集**：Playwright + BeautifulSoup
- **文档解析**：PyMuPDF + python-docx + 现有解析器
- **表格提取**：camelot-py（PDF表格）
- **视觉对比**：pixelmatch
- **文本分析**：jieba分词 + sklearn（关键词提取、主题分类）
- **任务队列**：Celery + Redis
- **数据库**：SQLite（开发）/ PostgreSQL（生产）

### 8.2 前端技术栈
- **框架**：Vue 3 + Vite（与现有平台一致）
- **UI组件**：Element Plus
- **图表**：ECharts（雷达图、趋势图、词云）
- **双栏对比**：基于现有PdfPreview组件扩展
- **富文本展示**：文章详情页渲染

### 8.3 核心数据流
```
竞品配置（网页/文档/公众号）
          ↓
    定时任务触发
          ↓
┌─────────┴─────────┐
↓                   ↓
网页抓取         公众号采集
│                   │
└─────────┬─────────┘
          ↓
    与上一版本对比
          ↓
    ┌─────────────┐
    │ 差异分析结果 │
    └──────┬──────┘
           ↓
┌──────────┴──────────┐
↓                     ↓
变化通知推送     AI智能分析
                        ↓
              改进建议 + 报告
```
