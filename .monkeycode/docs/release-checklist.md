# 文档发布前 Checklist

本清单以以下两个附件为发布前检查基准：

- 中文说明书写作 Checklist：`.monkeycode-tmp-files/152fab92-中文说明书写作Checklist-1.xlsx`
- 英文说明书写作 Checklist：`.monkeycode-tmp-files/bb261344-英文说明书写作Checklist-2.xlsx`

知识库管理中的发布前规范文件位于：`写作规范 > 说明书自检checklist`。

## 发布阻断项

- [ ] 修订历史表中的最新版本号高于上一版本号。
- [ ] 修订历史表中的最新修订日期晚于上一版本修订日期。
- [ ] 最新修订年份与当前发布年份一致，例如 `2026` 不能误写为 `2025`。
- [ ] 文档首页、前言、修订历史表、文件名中的年份和发布日期一致。
- [ ] 自动审核结果中仍存在 `待修改` 问题时，阻断发布。
- [ ] 安全、法规、适用范围、关键数值、单位、产品型号存在未确认问题时，阻断发布。

## 中文说明书发布前 Checklist

### 打印

- [ ] 骑马钉：不加封面时，页数为 4 的倍数。
- [ ] 胶装：不加封面时，页数为偶数。

### 前言

- [ ] 当年首版：写当年年份。
- [ ] 非当年首版：写为 `首版年份 - 当前版本年份`。
- [ ] 文中出现的商标需列明，且为上标格式，例如 `QubitTM`、`Qubit®`、`NextseqTM`、`DNBSEQTM`、`MGIEasyTM`。
- [ ] 统一日期格式，例如 `2022年7月21日`。

### Logo

- [ ] 封面 Logo：除部门特殊规定，中文说明书用“中文+英文”的 Logo，英文说明书用英文 Logo。
- [ ] 封底 Logo：要求高清，必要时根据文档配色调整二维码颜色。

### 图片

- [ ] 使用统一的注意图标。
- [ ] 尽量使用矢量图，勿从其他文档截图，确保图中无空格和段落标记，无变形。
- [ ] 标注线整齐一致，尽量朝一个方向排列，置于顶层。
- [ ] 标注文字需考虑易读性。

### 语言

- [ ] 表达简洁，避免重复累赘。
- [ ] 表达逻辑清晰，注意前后逻辑和主语是否正确。
- [ ] 使用书面语，避免口语化表达。
- [ ] 避免错别字。

### 内容排列格式

- [ ] 内容多且可以分类时，采用表格或列表形式。
- [ ] 只有 1 个注意事项时，使用“注意图标+正文内容样式”。
- [ ] 多个注意事项时，使用“注意图标+无序列表”。
- [ ] 注意事项尽可能放在具体操作后。
- [ ] 多个同层级内容并列出现时，使用无序列表。
- [ ] 操作步骤使用“引导句+有序列表”。
- [ ] 斜体适用于变量、交叉引用、网址链接。
- [ ] 标题句型统一为“动词+名词”，例如“准备试剂槽”。
- [ ] 表格中表头和表体内容的字体与大小分别保持一致。
- [ ] InDesign 中正确应用表格字符样式和单元格样式，必要时重新载入最新模板。
- [ ] 单位书写正确，例如 `2 mL/支×1`。
- [ ] 避免将 `μ` 写成 `U`。
- [ ] 商标、论文引用标注使用上标。
- [ ] 数字一行显示，避免断行；公式独立成行。
- [ ] 标题和内容尽量同页显示。
- [ ] 封面与封底不插入页码。
- [ ] 网址链接统一不加下划线，正文内斜体、独立成行。
- [ ] 字体字号需同类统一。
- [ ] 根据内容适当调整列宽与边距。

### 交叉引用

- [ ] 使用交叉引用样式，不加书名号，斜体。
- [ ] 引用完整的正式文件名。
- [ ] 需打印时，表格尽量不要离正文太远。
- [ ] 只以电子版发布时，直接交叉引用。
- [ ] 图表就在句子下方时，使用“如下表”“下图”等表达。
- [ ] 泛指引用无需使用交叉引用格式，例如“相关软件操作指南”。

### 标点符号

- [ ] 中文使用中文标点，英文使用英文标点。
- [ ] 货号后需加冒号，例如 `Cat. No.:`。
- [ ] 避免冒号与逗号混用。
- [ ] 避免整段话全部使用逗号。
- [ ] 一句话结束时必须有句号。
- [ ] 步骤、列表句末使用句号。
- [ ] 表示温度时，负号务必与数字同行显示。
- [ ] 避免多余空格。
- [ ] 数字与单位间需有空格。
- [ ] 尽量少用括号。
- [ ] 使用标准数学符号，例如 `×`、`≥`、`≤`。
- [ ] 步骤中的大于号仅用于表示按钮操作顺序。
- [ ] 除警告标志外，不使用感叹号。

### 图片

- [ ] 使用统一的注意图标。
- [ ] 尽量使用矢量图，勿从其他文档截图，确保图中无空格和段落标记，无变形。

- [ ] 使用书面语，避免口语化表达。
- [ ] 使用美式英语。
- [ ] 使用 Word 或类似拼写检查工具辅助检查拼写。
- [ ] 配置触摸屏时，点击界面按钮使用 `Tap`。

### 语法

- [ ] 句子时态前后一致。
- [ ] 尽量使用主动语态。
- [ ] 检查冠词缺失或冠词多余。
- [ ] 定冠词 `the` 用于众所周知或特指名词前。
- [ ] 不定冠词 `a/an` 用于首次出现或非特指名词前。
- [ ] 标题首个单词首字母大写，其余小写，并保持统一。
- [ ] 任务导向标题使用动词 `-ing` 形式。
- [ ] 每一步骤首句使用祈使句动词，并加上引导句。
- [ ] 使用简单、直接、有力的动词。
- [ ] 避免使用主观意向词，例如 `ask`、`decide`、`expect`、`see`、`think`、`want`、`refuse`。
- [ ] 避免使用语气弱或模糊的表达，例如 `have`、`make`、`give`、`based on`、`require to`、`enable`、`there are/is/will be/have been`、`here are/is`、`it is`。
- [ ] 不及物动词后接名词时，中间加介词。
- [ ] 避免使用 `It`、`This`、`That`、`These`、`Those`、`There are/is` 作模糊指代，改用真实主语或重写句子。
- [ ] 避免出现表示性别的人称，例如 `his` 和 `her`。
- [ ] 限定性定语从句使用 `that`。
- [ ] 非限定性定语从句使用 `which`，前面必须有逗号。
- [ ] 用 `who` 指代人。
- [ ] `Recommend` 相关表达使用 `It is recommended to use/that ...` 或 `... is/are recommended.`。
- [ ] 检查单复数形式是否正确。
- [ ] 若 `if` 从句的主语与主句主语一致，且从句主语后跟 `be`，可省略 `主语+be`。
- [ ] 检查内容是否缺失。
- [ ] 检查用词是否语义重复。

### 内容排列格式

- [ ] 界面词引用加粗。
- [ ] 内容多且可以分类时，采用表格或列表形式。

- [ ] 一个 Task 主题只描述一个任务。
- [ ] Task 标题使用动词词组或“How to ...”形式。
- [ ] 每个步骤操作命令清晰。
- [ ] 每个任务的步骤数量控制在 10 个以内。
- [ ] 合并过短步骤命令。
- [ ] 必要时，指明步骤是必须操作还是可选操作。
- [ ] 斜体适用于变量、交叉引用、网址链接。
- [ ] 操作类标题、图名、表格名统一为 `动词ing+名词`。
- [ ] 表格中表头和表体内容的字体与大小分别保持一致。
- [ ] InDesign 中正确应用表格字符样式和单元格样式，必要时重新载入最新模板。

- [ ] `μL` 中的 `L` 务必大写，避免与 `l`、`1` 混淆。
- [ ] 商标、论文引用标注使用上标。
- [ ] 数字一行显示，避免断行；公式独立成行。
- [ ] 标题和内容尽量同页显示。
