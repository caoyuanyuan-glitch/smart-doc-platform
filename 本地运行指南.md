# 智能文档审核平台 - 本地运行指南

## 环境要求

在开始之前，请确保已安装以下软件：

| 软件 | 版本要求 | 下载地址 |
|------|----------|----------|
| Python | 3.9+ | https://www.python.org/downloads/ |
| Node.js | 16+ | https://nodejs.org/ (推荐 LTS 版本) |

## 快速启动（Windows 命令提示符）

### 1. 克隆/下载代码

**方式 A：从 GitHub 下载 ZIP（推荐）**
1. 访问：https://github.com/caoyuanyuan-glitch/smart-doc-platform
2. 点击 `Code` → `Download ZIP`
3. 解压到桌面

**方式 B：使用 Git 克隆**
```bash
git clone https://github.com/caoyuanyuan-glitch/smart-doc-platform.git
cd smart-doc-platform
git checkout feature/document-review-and-convert
```

## 启动后端服务

### 步骤 1：打开命令提示符

按 `Win + R`，输入 `cmd`，回车

### 步骤 2：进入后端目录

```bash
cd Desktop\你的项目文件夹名\backend
```

> **注意**：路径中的文件夹名可能因下载方式不同而有所差异
> - ZIP 下载后可能是：`smart-doc-platform-main`
> - Git 克隆后可能是：`smart-doc-platform`

### 步骤 3：创建虚拟环境（仅第一次运行）

```bash
python -m venv venv
```

### 步骤 4：激活虚拟环境

```bash
venv\Scripts\activate
```

成功后会看到命令提示符前有 `(venv)` 标记

### 步骤 5：安装依赖（仅第一次运行）

```bash
pip install -r requirements.txt
```

> **如果安装太慢，可以使用国内镜像源：**
> ```bash
> pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
> ```

### 步骤 6：创建本地 AI 配置（图片描述生成必需）

在 `backend` 目录下复制模板文件：

```bash
copy runtime.env.example runtime.env
```

打开 `runtime.env`，至少填写以下配置：

```env
KIMI_API_KEY=你的KimiKey
KIMI_BASE_URL=https://api.moonshot.cn/v1
KIMI_MODEL=kimi-k2.6
```

> **说明：** `runtime.env` 不会提交到 Git。每台新电脑首次运行后端时都需要创建一次，后续更新代码可继续复用。

### 步骤 7：初始化数据库（仅第一次运行）

```bash
python init_data.py
```

### 步骤 8：启动后端服务

```bash
uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```

**成功标志：**
```
Uvicorn running on http://0.0.0.0:8000 (Press CTRL+C to quit)
Application startup complete.
```

## 启动前端服务

### 步骤 1：打开新的命令提示符窗口

**保持后端窗口开着！** 再打开一个新窗口

### 步骤 2：进入前端目录

```bash
cd Desktop\你的项目文件夹名\frontend
```

### 步骤 3：安装依赖（仅第一次运行）

```bash
npm install
```

> **如果安装太慢，可以使用国内镜像源：**
> ```bash
> npm config set registry https://registry.npmmirror.com

> ```

### 步骤 4：启动前端服务

```bash
npm run dev
```

**成功标志：**
```
VITE ready in XXX ms
→ Local: http://localhost:5173/
```

## 常见问题

### Q1：找不到 `python` 命令

**解决方法：**
```bash
# 试试 python3
python3 --version

# 或使用 py 命令
py --version
```

### Q2：`pip install` 安装失败

**解决方法：**
```bash
# 升级 pip
python -m pip install --upgrade pip

# 使用国内镜像
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

### Q3：`npm install` 安装太慢

**解决方法：**
```bash
# 使用淘宝镜像
npm config set registry https://registry.npmmirror.com

# 重新安装
npm install
```

### Q4：端口被占用

**后端换端口：**
```bash
uvicorn app.main:app --host 0.0.0.0 --port 8001 --reload
```

**前端换端口：**
```bash
npm run dev -- --port 5174
```

> 注意：换端口后，访问地址也要相应改变（如 http://localhost:5174）

### Q5：遇到 `ModuleNotFoundError`

**解决方法：**
```bash
# 重新安装依赖
pip install -r requirements.txt
```

### Q6：数据库报错

**解决方法：**
```bash
# 删除旧数据库（如果存在）
del app.db

# 重新初始化
python init_data.py
```

### Q7：浏览器显示空白页

**解决方法：**
1. 按 `F12` 打开开发者工具
2. 查看 Console 是否有红色错误
3. 刷新页面（尝试 Ctrl + Shift + R 强制刷新）

### Q8：图片描述生成返回兜底文本

**现象：** 页面提示“当前多模型图片分析链路未返回有效结果，结果来自本地兜底逻辑”。

**解决方法：**
```bash
# 检查本地运行参数文件是否存在
dir runtime.env

# 检查模型配置是否为支持图片输入的版本
findstr KIMI_MODEL runtime.env
```

确保 `runtime.env` 中使用的是：

```env
KIMI_MODEL=kimi-k2.6
```

修改后重启后端服务。

## 一键启动脚本（可选）

如果觉得每次启动麻烦，可以创建批处理文件：

### 创建后端启动脚本 `start-backend.bat`

```batch
@echo off
cd /d %~dp0backend
call venv\Scripts\activate
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
python init_data.py
start uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```

### 创建前端启动脚本 `start-frontend.bat`

```batch
@echo off
cd /d %~dp0frontend
call npm install
start npm run dev
```
